CONE百语翻译

CONE是一个在NLLB-1.3B的基础上，在开源机器翻译语料上进行训练得到的轻量级多语言翻译模型。其中总训练语料包括250亿平行数据，300+语言，20000+翻译语向。目前模型仍在持续优化中，我们选出了效果较好的200+语言进行内测，内测结束后会将全部模型和技术细节进行开源。

导言

世界上有上千种正在使用的语言，无论是开源模型还是商业翻译软件，目前覆盖的语言种类仍然十分有限。实际上，许多使用中的语言甚至没有任何可用的机器翻译系统。如何扩展语言边界是CONE项目致力于研究的关键问题。每种语言都有其独特的语法、词汇和表达方式，要让计算机能够准确地翻译这些语言，需要大量的语言数据、人工评测、计算资源等。一些小语种的使用者数量较少，扩展小语种存在大量的挑战。

相关产品比较

为了更好的理解和量化CONE的能力，以下是对相关产品的一些介绍：

  1. chatGPT类语言模型：
    优点：根据现有研究结果和我们的实验结果，chatGPT在德英大语向上性能可以逼近领先监督翻译模型效果，小语种时不时也有惊艳的翻译效果。
    缺点：支持语种数量比较有限；模型参数多，推理代价大；小语种统计支持相对。

  2. 商业翻译软件：
    优点：核心语向积累较多，用户体验好，翻译性能领先；
    缺点：大部分的商业软件仅支持几十种或者上百种语言，比如DeepL、Google等。百度是唯一支持200种语言的翻译产品（如果有疏漏，可以调整）；大规模调用需要花钱调用API接口；

  3. 开源翻译模型：
    优点：免费；可定制化；后续研究可以复用开源工作。
    缺点：翻译性能相对比较有限；比如领先开源机器翻译模型NLLB对非拉丁语系支持有限，中文翻译性能较差;性能较好的翻译模型比如NLLB-54B推理代价又较大。

以上是一些定性的介绍，具体的性能比较可以参考下文中的模型对比部分。

不同产品有不同的优缺点。机器翻译是积累比较多的垂直领域，chatGPT目前也仅只能在有限的语向上逼近SOTA有监督翻译结果。目前来看，垂直领域的小模型仍然是性价比比较好的选择。

但是由于chatGPT是通用模型，接近SOTA的翻译效果也展示了通用模型用于垂直领域的潜力，下一步我们将会结合机器翻译和开源的通用模型译进行垂直领域性能SOTA新探索。

对比chatGPT

chatGPT主要优势为英语相关的翻译，我们评测了chatGPT上英文的翻译效果并和CONE进行了比较。由于经费限制，我们选择了flores-101的前100句话进行翻译评测。评测方式统一采用了flores-101的spBLEU分数。chatgpt生成结果和CONE具体生成结果见chatgpt目录。可以看到：CONE在chatGPT的优势语种英语上大幅超过chatGPT的平均翻译效果，在英语到93种语言翻译、99种语言到英语翻译取得性能领先。

对比商业翻译软件

我们分别选择了国外知名商业系统(A)和国内知名商业翻译软件(B)进行了比较。商用系统在大语种上支持较好，不像chatGPT具有明显的英语倾向性。由于CONE的主要对标对象为开源系统，我们还比较了NLLB-1.3B模型进行效果对比。

为了更准确地衡量翻译效果，我们利用了中文评测人员优势，选择100条中文数据进行回译评测。100条数据来源包括：XX条的flors-101数据，XX条的最新体育、娱乐、财经新闻等数据。回译评测指通过将中文翻译成对应语言，再将对应语言翻译回中文，通过对比中文信息保留程度的方法进行比较。每个样本会同时分发给两位不同的标注员。标注员的平均相关性分数为XX，表明标注一致性比较高。评测分数包括0-5分。标注的其中0分表示不支持的语言或完全翻译不出来，1分表示仅能保留很少很少一部分单词，2分表示能保留一小部分内容，3分表示能保留大部分内容，4分基本上内容都可以还原，5分表示不仅内容可以保留，表达也非常流畅。比较结果可见商业翻译软件对比文件夹。

人工评测的性能结果为商业系统A平均好于CONE，CONE平均好于商业系统B。不同于商业系统背后复杂的处理逻辑，考虑到CONE背后仅为单一模型，并且CONE支持的语言更多，可以达到商业系统的分数仍然是一个比较吃惊的效果。

以下是和商业系统A的比较结果。除了中文外，共有117种重合语言，商业系统A独立支持17种，CONE独立支持120种小语种。在重合的语言中CONE翻译的平均分数为3.12分，商业系统A的翻译效果均分为3.64分。

和商业系统B相比，除了中文外，共有144种重合语言，商业系统B额外独立支持约56种语言，CONE额外独立支持93种语言。在重合语言中CONE效果高于商业系统B的有95/144种。重合语言的CONE模型均分为3.03，商业系统B的模型均分为2.55。

除此之外，我们还比较了起始模型NLLB-1.3B的翻译效果。由于NLLB-1.3B为起始模型，这里并不是公平比较，只是为了说明开源系统和商用系统之间存在的差距。除了中文外，CONE效果高于NLLB的有140种语言, CONE模型的均分为2.99，NLLB模型均分2.23。

说明：此处和商业产品比较仅是为了反映开源模型的大概能力。由于商业系统背后所用的技术、数据处理流程、使用的数据都不同于CONE，这里的比较只是为了观察CONE的大概翻译水平。由于评测数据有限，我们目前仅人工评测了100条，后续我们会推动更多更丰富的人工评测比较。

常见问题及解答

Q1：内测服务为什么会返回输入错误和生成内容错误？
为了避免翻译系统被用来翻译不当内容，我们调用了阿里云的敏感词过滤接口。如果出现误触发问题，请联系cone@x-venture.tech进行反馈。

Q2：为什么模型在小语种的翻译质量是不是出现翻车案例？
尽管我们优化了小语种的翻译能力，但是小语种的翻译性能和大语种的翻译性能还是有明显差距，因此出现小瑕疵等问题都是小语种仍然面临的问题。如果您觉得翻译效果不如预期，可以尝试百度或者Google等商业软件进行测试。

Q3：为什么对外服务还有预约？
由于内测服务资源有限，我们只能在同一时间段对有限的用户提供服务，敬请谅解。

Q4: 我是专业的小语种用户，发现问题怎么反馈？
小语种的评测问题目前除了自动评测还没有更好的解决方案，如果您懂小语种，欢迎联系cone@x-venture.tech。我们也正在积极招募小语种标注员和运营实习生。
